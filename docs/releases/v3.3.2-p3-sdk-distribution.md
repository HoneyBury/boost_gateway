# v3.3.2 P3 SDK 企业级封装记录

日期：2026-05-17

提交基线：`9973279`

本记录归档 P3 首批 SDK 企业级封装与分发质量收束。当前阶段不扩展服务端功能，重点是让 SDK 的版本、C++ package、C ABI 动态库、Python/C# 轻量封装和验证门禁形成闭环。

## 交付结论

P3 已完成 SDK 分发与 ABI 边界首批硬化：

- SDK CMake/package 版本统一到 `v4.1.0`。
- `boost_gateway_sdk_dll` C API 动态库纳入安装产物。
- C API 新增 `gsdk_version()`，用于语言封装运行时校验 native SDK 版本。
- C API 对空句柄、空字符串、负 timeout 和 C++ exception 做 ABI 边界保护。
- Python wrapper 暴露 `version()` 并在 native library 缺失时显式报错。
- C# wrapper 补齐 version、battle、echo 绑定。
- 新增 `scripts/check_sdk_distribution.py`，把 SDK 版本、导出符号、安装目标、语言包装覆盖面和可选构建产物纳入门禁。

## 验证命令

```bash
python3 -m py_compile scripts/check_sdk_distribution.py
python3 scripts/check_sdk_distribution.py --build-dir build/default --summary-path runtime/validation/p3-sdk-distribution-summary.json
cmake --build build/default --target boost_gateway_sdk boost_gateway_sdk_dll sdk_tests
ctest --test-dir build/default -R "CodecV4Test|ClientV4Test|Transport" --output-on-failure
python3 scripts/check_deploy_operability.py --build-dir build/default --summary-path runtime/validation/p2-deploy-operability-summary.json
```

## 当前限制

- SDK 仍面向当前 v1/v2 wire protocol；typed envelope/proto/gRPC SDK 入口属于后续通信契约阶段。
- Python/C# wrapper 当前为薄绑定，不替代正式包管理发布。
- C API 错误码仍使用当前 SDK result/error 字段，后续可进一步标准化跨语言错误枚举。
