# v3.3.2 P0 Production Deployment Closure

Date: 2026-05-17

## Scope

This pass closes the P0 production deployment surface for the next stabilization phase. It does not add new business modules. The goal is to make the current deployment assets auditable, repeatable, and aligned with the real runtime topology.

## Changes

- Added `docs/production-deployment-runbook.md` as the production deployment fact source.
- Pinned Kubernetes core application images to `v3.3.2` and aligned `app.kubernetes.io/version`.
- Changed Kubernetes application image pull policy to `IfNotPresent` for pinned tags.
- Fixed `scripts/deploy_k8s.py --dry-run` to parse manifests offline with `kubectl create --dry-run=client --validate=false`.
- Extended `scripts/check_deploy_operability.py` to block:
  - Kubernetes app version drift.
  - Floating `latest` tags for core app manifests.
  - Missing pinned Kubernetes images.
  - Prometheus/backend scrape scope drift.
  - Undocumented gateway `/health` readiness limitations.
- Corrected environment docs so Prometheus is documented as scraping gateway `/metrics` only.
- Linked the production deployment runbook from the docs index and deployment guide.

## Remaining Boundaries

- A real cloud or cluster rollout was not performed in this pass; `deploy_k8s.py --dry-run` validates manifest parsing only.
- gateway `/health` remains a liveness stub. Production readiness must be validated with SDK full-flow or P6 production evidence.
- Backend HTTP metrics are not implemented; Prometheus must scrape gateway `/metrics` only until a backend management surface or sidecar exporter is added.
