# v3.3.2 P4 SDK 企业级运行语义收束记录

日期：2026-05-17

本记录归档下一大阶段 P4 的第一轮交付：在既有 SDK 分发和业务闭环基础上，补齐生产客户端接入所需的 heartbeat、disconnect lifecycle、push 分发约束、reconnect 语义和跨语言版本诊断。

## 交付结论

P4 已完成 SDK 运行语义收束：

- C++ SDK `start_heartbeat()` / `stop_heartbeat()` 不再是空实现，会按间隔发送 `kHeartbeatRequest` 并验证 `kHeartbeatResponse`。
- SDK 内部用 I/O 锁串行化 heartbeat 与同步业务请求，避免并发读写同一个 TCP socket。
- heartbeat 失败会关闭连接并触发 `on_disconnect` callback；主动 `disconnect()` 仍保持幂等且不触发意外断开回调。
- `connect()` 会先清理旧 socket，再建立新连接，支撑 `disconnect()` -> `connect()` -> `login()` 的 reconnect 流程。
- 未连接、发送失败、超时和无效响应会写入 SDK result 的 `error_code` / `error_message`。
- C ABI 新增 `gsdk_start_heartbeat()` / `gsdk_stop_heartbeat()`。
- Python wrapper 增加 `BOOST_GATEWAY_SDK_LIBRARY` 加载入口、加载失败诊断、native library 路径查询和主版本校验。
- C# wrapper 增加 native 主版本校验、allocation failure 诊断和 heartbeat 控制方法。
- SDK 文档明确同步 API、回调线程/调用边界、disconnect 生命周期、heartbeat/reconnect/push 语义。

## 当前边界

- 当前 SDK 仍是同步 API；回调内不要递归调用同一个 `SdkClient` 的同步请求。
- `on_disconnect` 目前由 heartbeat failure 驱动，主动 `disconnect()` 不触发。
- Python/C# wrapper 仍是轻量绑定，不替代正式包管理发布；但已具备 native library 版本诊断和加载失败提示。
