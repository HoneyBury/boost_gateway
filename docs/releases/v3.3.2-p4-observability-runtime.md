# v3.3.2 P4 Observability Runtime Gate

Date: 2026-05-17

## Scope

P4 already had unit and integration coverage for rate limiting, trace propagation, OTel exporter behavior, backend RED metrics, gateway metrics rendering, and audit evidence. This pass adds a runtime-facing gate for the production-like HTTP observability path.

## Changes

- Added `scripts/verify_gateway_observability_runtime.py`.
- The new gate starts a real `v2_gateway_demo` process with HTTP management enabled.
- It runs `sdk_full_flow_client` to generate real TCP sessions, outbound dispatches, battle pushes, surrender settlement, and reconnect traffic.
- It verifies:
  - `/health` responds with a valid health status.
  - `/ready` reports `ready: true` in local demo mode.
  - `/metrics` exposes Prometheus session and dispatch counters.
  - `/metrics/json` reports positive runtime counters after SDK traffic.
  - `/metrics/diagnostics/json` preserves the diagnostics JSON shape.
- `scripts/verify_observability_gate.py` now supports `--include-runtime-http` for fixed runners.
- PowerShell wrapper, release checklist, runtime playbook, and reliability matrix were updated.

## Validation

```bash
python3 -m py_compile scripts/verify_observability_gate.py scripts/verify_gateway_observability_runtime.py
python3 scripts/check_reliability_matrix.py

python3 scripts/verify_observability_gate.py --build-dir build/default --configuration Debug --skip-build --include-runtime-http --summary-path runtime/validation/p4-observability-gate-summary.json
```

## Notes

- Local demo `/health` may report `warn` when no external backend processes are registered. The runtime gate accepts `pass`, `ok`, or `warn` for liveness but still requires `/ready` to report `ready: true`.
- The runtime HTTP gate binds local TCP and HTTP ports and should run on a fixed runner or local machine with loopback bind permission.
