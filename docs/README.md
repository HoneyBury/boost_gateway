## 文档使用建议

1. 开始新阶段前先看《高性能游戏服务器框架规划》。
2. 日常开发按《工程开发规范》执行。
3. 每次较大迭代结束后更新《开发日志模板与阶段记录》。
4. 所有分支流转、提交信息和合并动作遵循《Git 工作流与提交规范》。
