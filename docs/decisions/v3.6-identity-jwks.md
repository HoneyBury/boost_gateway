# ADR: v3.6 外部身份 JWKS 与多 kid

- 决策 ID：`v36-identity-jwks`
- Status：`accepted_for_implementation`
- Target Minor：`v3.6.0`
- Default Activation：`blocked_until_gates_pass`
- 日期：2026-07-21

## 背景

生产 Login Backend 当前只验证外部身份提供方签发的 RS256 JWT，要求静态
public key、issuer、audience 和 `exp`，并禁止对称 secret、私钥、本地注册、
guest login 和 refresh token 签发。`JwtHeader` 虽然声明了 `kid`，验证器实际
只读取 `alg`，配置也只能表达单个 PEM，因此现有生产配置无法在两个公钥并存
时完成无中断轮换。

本项目仍然是令牌验证方，不成为账户、凭据或 refresh token 的事实源。

## Decision / 决策

在 `v3.6` 保留现有静态 PEM 路径，并分两步增加外部 key resolution：

1. 先引入不可变 `KeySet` / `KeyResolver` 边界和静态多 `kid` key ring。多 key
   模式要求 JWT header 携带非空 `kid`；未知、重复或歧义 `kid` 一律拒绝。
   单静态 PEM 在兼容窗口内继续接受无 `kid` 的 RS256 token。
2. 再增加显式配置的 JWKS URI。JWKS 在后台单飞刷新并以不可变快照原子替换，
   登录请求不得执行无界网络 I/O。未知 `kid` 可以触发限速异步刷新，但当前
   token 仍然 fail closed，调用方重试后才能使用新 key。

首个实现只接受 RSA/RS256 key。JWT `alg`、JWK `kty`、`use`、`key_ops` 必须
一致，不允许算法回退。issuer、audience、subject、`exp` 继续强制验证；到期
时刻按 `now >= exp` 拒绝，畸形或类型错误 claim 必须稳定返回验证错误而不是
抛出到请求线程。

JWKS 获取策略必须满足：

- 生产 URI 只允许 HTTPS，测试仅允许显式启用的 loopback HTTP；禁止重定向，
  host 必须与配置的 allowlist 精确匹配，避免 SSRF。
- connect/read/total timeout、响应字节数、key 数量、单 key 大小、刷新频率和
  stale grace 均有上限。
- 有可用快照时，刷新失败可在有界 stale grace 内继续使用旧 key，并暴露
  freshness/refresh 指标；超过 grace 后拒绝依赖过期快照的 token。
- 仅配置 JWKS 且启动时没有可用 key 时，Login Backend 启动失败。首版不把
  JWKS 缓存持久化到磁盘，避免把未定义的信任状态跨重启延续。
- 日志和指标不得记录完整 token、PEM 或 JWK 私密材料。

## Compatibility / 兼容策略

- `auth.mode=external-jwt` 语义保持为外部 RS256 验证方，不自动切换 JWKS。
- 现有 `auth.jwt_public_key_pem` / `V2_LOGIN_JWT_PUBLIC_KEY` 在整个 `v3.6.x`
  保持支持，作为回滚路径。
- 新 key ring/JWKS 配置与 secret/private key 互斥。生产仍拒绝本地签名材料。
- 单 PEM 到 key ring 的迁移先把旧 key 和新 key 同时部署，再切换 IdP 签发
  `kid`，最后在令牌最大寿命加时钟偏差之后移除旧 key。
- `token_validate`、login 和 session 相关入口必须使用同一 resolver snapshot，
  且 token subject 必须与请求的 `user_id` 一致。

## Migration Window / 迁移窗口

1. `v3.6.0` 首批实现静态单 PEM + 静态多 `kid` 双路径，默认仍使用现有单 PEM。
2. JWKS 缓存和负路径门禁完成后，部署方可显式切到 JWKS；混合部署期间保留
   静态 key ring 作为独立回滚配置，不允许在一次请求中隐式降级选钥。
3. 静态单 PEM 在 `v3.6.x` 不弃用。是否在 `v3.7` 以后收缩必须另写 ADR，
   不能由本决策自动移除。

## Release Assets / 发布资产

- 更新后的 production login 配置模板和配置 schema/环境变量说明；示例不得
  包含真实 key 或可访问的生产 JWKS URI。
- 身份配置与轮换 runbook、错误码/指标说明及安全发布门禁 summary。
- JWKS/key ring 本身不是发布包资产，不得写入 tarball、SBOM 或测试日志。

## Validation Matrix / 验证矩阵

| 层级 | 必须覆盖 |
|---|---|
| Unit | 单/多 key 成功；missing/unknown/duplicate `kid`；alg/kty/use/key_ops 不匹配；畸形 header/claim；`now == exp`；issuer/audience/subject |
| Cache | TTL、stale grace、原子快照、单飞刷新、未知 `kid` 刷新限速、超时、超大响应、过多 key、失败后 fail closed |
| Integration | Login Backend 静态 key 轮换、JWKS 新旧 key 并存、IdP 不可达、过期快照、所有 token validation 路径 subject 一致性 |
| Security gate | 生产禁止 HS256/私钥、本地身份操作；HTTPS/allowlist/redirect 配置；日志不泄漏 token/key |
| Operations | key freshness、最后成功刷新、失败数、unknown kid 拒绝数可观测；轮换与回滚演练有结构化结果 |

## Activation and Rollback / 激活与回滚门槛

只有 unit/integration/security/operations 矩阵全部通过，且生产配置显式选择
key ring 或 JWKS 时才能启用。不得仅因 JWKS endpoint 可访问就调整默认配置。

回滚时先恢复仍有效的静态 key ring，再关闭 JWKS；不能回滚到 HS256、本地
private key 或跳过 issuer/audience/expiration。配置错误、无可用 key、缓存超过
stale grace 时都必须拒绝启动或拒绝 token。

## Out of Scope / 不支持范围

- 不实现账户持久化、密码存储、注册、guest 身份或 refresh token 签发/撤销。
- 不支持 OIDC discovery 自动改写 issuer/JWKS URI，不支持任意 URL 跟随跳转。
- 不在本决策中增加 ES256/EdDSA、远程 introspection 或每请求同步拉取 key。

## Manifest Contract

- Compatibility Window: static RS256 PEM remains supported throughout v3.6.x; removal or deprecation requires a later ADR
- Migration Window: deploy a static multi-kid key ring before enabling bounded JWKS refresh, and retain the static key ring as the independent rollback configuration
- Release Asset: production identity configuration schema
- Release Asset: key rotation and rollback runbook
- Release Asset: security gate and refresh observability summary
- Validation Gate: JWT key selection and malformed claim unit tests
- Validation Gate: bounded JWKS cache, refresh and failure integration tests
- Validation Gate: production identity security and rotation drill gates
