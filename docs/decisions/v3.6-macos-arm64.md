# ADR: v3.6 macOS ARM64 平台支持

- 决策 ID：`v36-macos-arm64`
- Status：`accepted_for_implementation`
- Target Minor：`v3.6.0`
- Default Activation：`blocked_until_gates_pass`
- 日期：2026-07-21

## 背景

当前发布与生产证据以 Ubuntu Linux x64 为准。仓库可在本机 macOS ARM64 进行
部分构建和测试，但 runner matrix 仍把 macOS 标为 smoke target，没有维护的
Conan ARM64 profile/lockfile、独立 CI summary、install consumer 或发布资产。

macOS 不能提供 Linux 的 affinity、`/proc`、Docker/kind 和 fixed-runner 容量
语义，因此平台支持与生产容量声明必须分开。

## Decision / 决策

- `v3.6` 将 macOS ARM64 定义为 build/test/install supported、capacity not
  claimed 的平台。Linux x64 继续是服务端生产 readiness、长稳和容量事实源。
- 新增独立 macOS ARM64 Conan profile、lockfile、CMake preset 和 workflow job。
  依赖必须来自锁定图，不允许因 Homebrew 或 runner 预装包偶然成功。
- smoke workflow 覆盖 Release configure/build、非 Linux 专属 unit/integration、
  CMake install、C++ SDK package consumer、C ABI 动态加载和有界 loopback
  full-flow。Linux-only 测试必须按能力显式跳过并在 summary 中列出原因。
- 发布 `boost-gateway-<version>-macos-arm64.tar.gz` 时，同时发布 checksum、SPDX
  SBOM 和 provenance。该资产面向命令行/服务端开发，不是 macOS GUI app；首批
  不承诺 Apple notarization。未签名/未 notarize 的边界必须出现在 release notes。
- wheel/NuGet 的 `osx-arm64` 变体必须复用相同 native build identity，且只有本
  ADR 的 package consumer/full-flow 门禁通过后才能加入 SDK supported matrix。

## Compatibility / 兼容策略

- 不修改 Linux 默认 preset、release workflow 或 fixed-runner 证据语义。
- 公共 C++/C ABI 在 Linux x64 与 macOS ARM64 保持同一 API/ABI source contract；
  平台文件名、RPATH/install_name 和动态加载规则可以不同，但必须文档化。
- 平台特定代码通过清晰能力边界隔离，不在通用代码中散布 runner 名称判断。
- macOS smoke 失败阻断 macOS 资产和 `osx-arm64` SDK 包，不反向伪造 Linux
  capacity 失败；若发布清单承诺 macOS 资产，则该 release 必须整体阻断。

## Migration Window / 迁移窗口

1. 先落地 profile/lockfile/preset 和手动 smoke workflow，不发布资产。
2. 连续候选在 clean macOS ARM64 runner 完成 build/test/install/package
   consumer 后，生成未发布 tarball 并验证 layout/RPATH/SBOM。
3. 在 `v3.6.0` 候选冻结时把 macOS 资产纳入 release manifest；线上独立下载
   验证通过后才把平台状态从 candidate 改为 supported。
4. 若资产门禁不稳定，回退为 smoke-only，并从该 release 的资产承诺中移除，
   不得上传未经验证的占位包。

## Release Assets / 发布资产

- macOS ARM64 runtime/C++ SDK tarball、SPDX SBOM、SHA-256 和 build provenance。
- 平台 summary，记录 runner image、OS/Xcode/Apple Clang/CMake/Conan、架构、
  lockfile hash、测试跳过清单和 package consumer 结果。
- 兼容矩阵与 release notes，明确无容量、Docker/kind 和 notarization 承诺。

## Validation Matrix / 验证矩阵

| 层级 | 必须覆盖 |
|---|---|
| Toolchain | ARM64 原生进程；锁定 Conan 图；Apple Clang/CMake 版本；禁止误产 x86_64/Rosetta artifact |
| Build/test | Release 全构建；适用 CTest；loopback gateway/backend/SDK smoke；资源释放无悬挂 |
| Install | tar layout、Mach-O arch、install_name/RPATH、动态依赖、CMake `find_package` consumer、C ABI load |
| Published asset | checksum、SPDX、provenance、独立下载解压和无源码最小运行 |
| Regression | Linux mainline/full test 与 fixed-runner gates 继续通过，macOS 条件代码不改变 Linux 行为 |

## Activation and Rollback / 激活与回滚门槛

只有 build/test/install 和线上资产矩阵在同一候选 SHA 通过，才能发布并宣称
macOS ARM64 supported。单次本机成功或已有 macOS summary 不构成发布证据。

回滚时移除尚未发布的 macOS 资产承诺并保持 Linux release；已经发布的不可变
资产不覆盖，使用后续修复版本和 release advisory。不得移动 tag。

## Out of Scope / 不支持范围

- 不声明 macOS 容量、2h/8h soak、CPU affinity、Docker/kind/Operator 或生产
  readiness 等价于 Linux。
- 不承诺 macOS x64、universal binary、Windows、iOS、Apple code signing、
  notarization 或 symbol server。

## Manifest Contract

- Compatibility Window: platform remains candidate until same-revision build, install and published-asset gates pass; Linux remains the capacity fact source
- Migration Window: land the toolchain and smoke profile first, validate an unpublished archive second, and add the immutable release asset only at candidate freeze
- Release Asset: macOS ARM64 runtime and C++ SDK archive
- Release Asset: SPDX SBOM, checksums and provenance
- Release Asset: platform toolchain and package-consumer summary
- Validation Gate: native ARM64 Conan and Release build
- Validation Gate: bounded CTest, SDK and loopback full-flow smoke
- Validation Gate: Mach-O install and published-asset consumer verification
