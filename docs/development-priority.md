# 开发优先级看板

## 1. 维护规则

这个文档用于长期维护当前项目的开发优先级，要求：

- 每完成一项核心任务，就同步更新状态。
- 每次调整技术方向或优先级顺序，都同步更新原因。
- 每个条目至少保留：
  - 当前状态
  - 目标说明
  - 产出物
  - 阻塞项或依赖项
- 状态约定：
  - `todo`
  - `doing`
  - `done`
  - `blocked`

## 2. 当前优先级

### P0

#### 2.1 状态管理拆分

- 状态：`done`
- 目标：把 `SessionManager` 中的房间态、战斗态拆分为独立管理器，降低耦合。
- 产出物：
  - `SessionManager`
  - `RoomManager`
  - `BattleManager`
- 依赖：无
- 备注：这是后续广播、退出房间、结算、重连恢复的基础。

#### 2.2 协议统一化

- 状态：`done`
- 目标：统一请求序号、错误码和响应结构，避免业务层继续用裸字符串协议。
- 产出物：
  - `request_id`
  - `error_code`
  - 统一回包规则
  - 协议测试更新
- 依赖：网络层和示例程序同步更新
- 备注：这是后续客户端联调和问题定位的基础。

#### 2.3 消息限频中间层

- 状态：`done`
- 目标：在网关层增加按连接维度的消息限频，防止异常请求冲击业务线程池。
- 产出物：
  - `rate_limit middleware`
  - 对应测试
- 依赖：`MessageDispatcher` 中间层机制
- 备注：后续可继续扩展为按消息号和按用户维度限频。

### P3

#### 2.10 可观测性升级

- 状态：`done`
- 目标：把当前日志型指标升级为 Prometheus 或等价监控输出。
- 产出物：
  - `GatewayMetricsExporter`
  - Prometheus 文本导出
  - JSON 指标快照导出
  - 对应单元测试
- 依赖：`GatewayMetrics`、`SessionManager`、`RoomManager`、`BattleManager`
- 备注：当前先落地为文件导出，后续可继续挂 HTTP `/metrics`。

#### 2.11 压测体系扩展

- 状态：`done`
- 目标：补齐登录风暴、广播压测、慢客户端压测和恶意包压测。
- 产出物：
  - JSON 压测配置
  - `echo` / `invalid_token` / `slow_echo` 场景
  - 压测汇总统计
- 依赖：`examples/pressure`
- 备注：广播压测和恶意包压测当前先留在场景扩展点，后续继续补齐战斗广播类压测。

#### 2.12 后端持久化集成

- 状态：`done`
- 目标：逐步接入数据库、缓存和账号服务。
- 产出物：
  - `JsonFileTokenValidator`
  - 鉴权提供方配置切换
  - 外部用户数据文件
  - 对应单元测试
- 依赖：`TokenValidator` 抽象、配置系统
- 备注：当前先完成本地文件后端接入，后续可继续替换为 HTTP / RPC 账号服务。
