# 开发优先级看板

## 1. 维护规则

这个文档用于长期维护当前项目的开发优先级，要求：

- 每完成一项核心任务，就同步更新状态。
- 每次调整技术方向或优先级顺序，都同步更新原因。
- 每个条目至少保留：
  - 当前状态
  - 目标说明
  - 产出物
  - 阻塞项或依赖项
- 状态约定：
  - `todo`
  - `doing`
  - `done`
  - `blocked`

## 2. 当前优先级

### P0

#### 2.1 状态管理拆分

- 状态：`done`
- 目标：把 `SessionManager` 中的房间态、战斗态拆分为独立管理器，降低耦合。
- 产出物：
  - `SessionManager`
  - `RoomManager`
  - `BattleManager`
- 依赖：无
- 备注：这是后续广播、退出房间、结算、重连恢复的基础。

#### 2.2 协议统一化

- 状态：`done`
- 目标：统一请求序号、错误码和响应结构，避免业务层继续用裸字符串协议。
- 产出物：
  - `request_id`
  - `error_code`
  - 统一回包规则
  - 协议测试更新
- 依赖：网络层和示例程序同步更新
- 备注：这是后续客户端联调和问题定位的基础。

#### 2.3 消息限频中间层

- 状态：`done`
- 目标：在网关层增加按连接维度的消息限频，防止异常请求冲击业务线程池。
- 产出物：
  - `rate_limit middleware`
  - 对应测试
- 依赖：`MessageDispatcher` 中间层机制
- 备注：后续可继续扩展为按消息号和按用户维度限频。

### P1

#### 2.4 完整登录链路

- 状态：`done`
- 目标：从字符串登录升级到 token 校验、顶号策略和登录上下文初始化。
- 产出物：
  - token 校验接口
  - 登录上下文
  - 重复登录策略

#### 2.5 房间系统独立化

- 状态：`done`
- 目标：补齐创建房间、离开房间、房主、准备状态、房间广播。
- 产出物：
  - 房间生命周期管理
  - 房间广播
  - ready 状态

#### 2.6 战斗系统独立化

- 状态：`done`
- 目标：补齐战斗上下文、输入路由、状态广播、结算。
- 产出物：
  - battle context
  - 输入分发
  - 状态同步
