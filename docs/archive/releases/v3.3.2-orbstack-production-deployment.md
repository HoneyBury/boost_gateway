## Production Fixes Applied

- Docker runtime ports now default to localhost for management, monitoring, Grafana, Redis host publish, Redis exporter, and Alertmanager; only gateway `9201` remains externally bound by default.
- Grafana default password is now environment-driven and no longer hardcoded as `admin/admin`.
- Docker builders now install `libssl-dev`, because the top-level CMake configure requires OpenSSL headers and libraries.
- Docker build contexts now copy `sdk/`, because the top-level CMake project includes `add_subdirectory(sdk)`.
- Docker CMake builds now use the `release` preset and bounded parallelism (`--parallel 2`) to reduce OrbStack build instability on local production rehearsal while producing production-grade binaries.
- Gateway and backend runtime images copy `libhiredis*.so*` and run `ldconfig`, because linked binaries require the hiredis shared library at runtime.
- Compose Redis host publishing defaults to `6380` to avoid colliding with a developer or machine-level Redis on host `6379`.
- Docker Compose sets `CONFIG_PATH=/app/config/environments/docker/gateway.json`, so gateway hot reload resolves backends by Compose service name instead of host loopback.
- Gateway TCP ingress now listens on `0.0.0.0:9201`, allowing Docker-published ports and Compose-network clients to reach the production ingress.
- Compose now includes Redis exporter and Alertmanager, and exposes cAdvisor through an optional `host-observability` profile.
- The SDK full-flow example now uses unique room ids per run and accepts both legacy text battle settlement pushes and current bridge-mode JSON `battle_finished` pushes.

## Operational Notes

- The gateway `/health` endpoint can report `warn` when the service registry has no dynamic registrations. In this Compose deployment, actual routing uses static backend configs loaded from `config/environments/docker/gateway.json`; business closure must therefore include the SDK full-flow check, not only `/health`.
- Docker images now use the CMake `release` preset. A later release-hardening pass can still add a shared builder/base image to reduce rebuild time.
- Stop the local production rehearsal with:

```bash
docker compose -f env/docker/docker-compose.yml down
```
