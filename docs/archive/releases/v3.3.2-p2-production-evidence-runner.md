# v3.3.2 P2 生产证据固定 Runner 收束记录

日期：2026-05-17

本记录归档下一大阶段 P2 的交付：把 P6 聚合门禁从“本地或人工命令能跑”推进到“固定 runner 可预检、可触发、可渲染、可归档”。

## 交付结论

P2 已完成 runner contract 收束：

- `production-evidence.yml` 支持 JSON runner 输入，避免多个 self-hosted label 被当成单一字符串。
- 生产证据 workflow 增加固定 runner concurrency、12h job timeout、preflight summary 归档和 GitHub Step Summary 渲染。
- 固定 runner 预检输出 `runtime/validation/fixed-runner-preflight-summary.json`，区分工具链缺失、Redis 不可达、kind/Docker 不可用和业务门禁失败。
- Workflow 已覆盖 Redis live、Operator kind、settlement replay、runtime HTTP observability、release baseline 和 capacity baseline 的显式输入。
- 新增 `docs/production-evidence-runner.md`，记录 runner label、场景输入、失败归因和 artifact 归档标准。

## 验证结果

本机执行了有界 P2 证据验证：

```bash
python3 scripts/check_fixed_runner_environment.py \
  --profile production-evidence \
  --build-dir build/default \
  --summary-path runtime/validation/fixed-runner-preflight-summary.json

python3 scripts/verify_production_evidence_gate.py \
  --build-dir build/default \
  --skip-build \
  --summary-path runtime/validation/production-evidence-summary.json
```

通过标准：

- `runtime/validation/fixed-runner-preflight-summary.json` 中 `passed=true`。
- `runtime/validation/production-evidence-summary.json` 中 `passed=true`。
- `runtime/validation/p6-stability-soak-summary.json`、`p6-data-recovery-summary.json`、`p6-specialized-e2e-summary.json` 均通过。

## 固定 Runner 完整验证要求

生产候选完整归档时，应在 self-hosted runner 上按需启用：

- `include_redis_live=true`
- `include_operator_kind=true`
- `include_settlement_replay=true`
- `include_observability_runtime=true`
- `include_release_baseline=true`
- 容量专项使用 `include_capacity_baseline=true`、`perf_repetitions=3`、`step_timeout_seconds=1800`

如果 preflight 失败，不计为业务回归；必须先修复 runner 环境。如果 preflight 通过但 P6 子 summary 失败，按 `production-evidence-summary.json` 的 `failed_category` 与 `failed_step` 定位。
