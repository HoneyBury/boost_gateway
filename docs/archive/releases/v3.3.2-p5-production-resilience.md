# v3.3.2 P5 长稳故障回滚收束记录

日期：2026-05-17

本记录归档当前生产稳定化大阶段的 P5：长稳、故障注入与回滚演练。目标是把“系统能启动”推进到“系统在 bounded soak、依赖故障、恢复路径、runtime 观测和 Kubernetes 回滚/删除路径中具备可重复证据”。

## 本轮交付

- 新增 `scripts/verify_production_resilience_gate.py` / `.ps1`，作为 P5 统一聚合入口。
- 默认 gate 保持有界：固定 runner 预检、stability soak、data recovery、Redis/Raft/Operator failure-path 专项。
- 固定 runner 可显式追加：
  - `--include-redis-live`：真实 Redis persistence/event-store/service live gates。
  - `--include-operator-kind`：Operator kind status smoke 与 control-plane kind gate。
  - `--include-runtime-http`：真实 gateway HTTP `/health`、`/ready`、`/metrics*` 验证。
  - `--include-release-baseline` / `--include-capacity-baseline`：release/capacity 性能回归证据。
- `scripts/check_fixed_runner_environment.py` 新增 `production-resilience` profile，区分 bounded 默认验证与 2h/8h 固定 runner 长项。
- 新增 `.github/workflows/production-resilience.yml`，支持手动固定 runner 触发、参数化真实依赖、Step Summary 渲染和 artifact 归档。
- `docs/reliability-matrix.md` 新增 `production_resilience_gate` 场景，绑定 P5 证据链。
- `docs/fixed-runner-playbook.md` 和 `docs/v3-release-checklist.md` 补齐 P5 执行命令、通过标准和阻断项。

## 验证入口

默认本地有界验证：

```bash
python3 scripts/verify_production_resilience_gate.py \
  --build-dir build/default \
  --configuration Debug \
  --skip-build \
  --summary-path runtime/validation/p5-production-resilience-summary.json
```

固定 runner 真实依赖验证：

```bash
python3 scripts/verify_production_resilience_gate.py \
  --build-dir build/default \
  --skip-build \
  --soak-profile short \
  --include-redis-live \
  --include-runtime-http \
  --summary-path runtime/validation/p5-production-resilience-live-summary.json

python3 scripts/verify_production_resilience_gate.py \
  --build-dir build/default \
  --skip-build \
  --include-operator-kind \
  --kind-timeout-seconds 1200 \
  --summary-path runtime/validation/p5-production-resilience-kind-summary.json
```

## 边界

- 默认 P5 gate 不承诺完成 2h/8h soak；长时长由固定 runner 扩大 `--soak-profile`、timeout 和外部执行窗口承担。
- 默认 P5 gate 不要求 Docker/kind 或 Redis live；相关真实依赖通过显式参数启用，并由 preflight summary 标出环境缺口。
- 当前 Operator kind 路径覆盖 rollout status、CR status conditions、六组件 status 和 sample CR 删除；更完整的业务级 rollback/probe E2E 仍作为固定 runner 长项继续沉淀。
