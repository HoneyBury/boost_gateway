# v3.3.2 N0-N2 生产数据沉淀与监控闭环收束记录

日期：2026-05-18

本文档记录 `../plans/production-stabilization-roadmap.md` 中 N0-N2 的首轮收束结果，目标是把固定 runner 证据自动化、性能证据索引和监控 SLO/告警闭环真正接入现有工程入口，而不是只保留规划描述。

## N0 固定 Runner 与证据自动化常态化

已完成：

- `release-baseline.yml`、`specialized-e2e.yml` 已对齐 JSON runner 输入，与 `production-evidence.yml` / `production-resilience.yml` 保持一致。
- 固定 runner 预检 summary 已统一归档到 `runtime/validation/*preflight*.json`。
- `scripts/render_validation_summary.py` 已支持统一渲染 `summary_version`、`environment`、`artifacts`、`overall_pass`。
- `collect_release_baseline.py`、`verify_specialized_e2e.py`、`verify_production_resilience_gate.py`、`verify_production_evidence_gate.py`、`verify_observability_gate.py`、`verify_stability_soak.py`、`check_fixed_runner_environment.py` 已统一到 N0 summary 契约。
- `docs/fixed-runner-playbook.md`、`docs/production-evidence-runner.md`、`docs/reliability-matrix.md` 已补固定 runner 证据矩阵与失败归因口径。

本地验证：

- `python3 scripts/check_fixed_runner_environment.py --profile release-baseline --build-dir build/default --summary-path runtime/validation/n0-release-baseline-preflight-summary.json`
- `python3 scripts/check_fixed_runner_environment.py --profile specialized-e2e --build-dir build/default --summary-path runtime/validation/n0-specialized-preflight-summary.json`
- `python3 scripts/check_fixed_runner_environment.py --profile observability --build-dir build/default --summary-path runtime/validation/n0-observability-preflight-summary.json`
- `python3 scripts/verify_specialized_e2e.py --build-dir build/default --skip-build --profile raft-ha --summary-path runtime/validation/n0-specialized-raft-ha-summary.json`

结果：

- 三个 preflight summary 均 `overall_pass=true`
- `n0-specialized-raft-ha-summary.json` `overall_pass=true`

## N1 长稳压测与容量基线

已完成：

- `docs/performance-baseline.md` 已补 N1 证据索引，明确 baseline、capacity、bounded soak、business-flow perf、docker production snapshot 的入口和产物。
- 性能边界已明确：capacity 失败可作为边界证据归档，但不得作为 baseline 通过线。
- 固定 runner 相关文档已明确 2h/8h soak、5K/10K capacity 和 battle-500 仍需继续沉淀固定机器实测数据。

当前结论：

- N1 的“证据口径和归档入口”已完成收束。
- N1 的“长稳与容量真实数据沉淀”仍需后续在固定 runner 或独占机器继续执行，不在本次静态交付内冒进宣称完成。

## 边界

- 当前 route latency 已从 `gateway_backend_*_avg_latency_us` 趋势升级为 `gateway_backend_route_latency_us_bucket/_sum/_count` 与 per-service P50/P90/P99 gauge。
- 后端服务仍未直接暴露 HTTP `/metrics`，N2 文档和脚本已明确禁止误宣称。
- 2h/8h soak、5K/10K capacity 和 battle-500 长稳复测仍属于后续 N1/N2 深化项。
