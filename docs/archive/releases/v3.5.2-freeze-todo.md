## 第二台 Runner：正式资产复验

在 `v3.5.2` tag publish 成功后，于第二 runner 的 `v3.5.2` detached checkout 根目录
执行。资产必须下载到新建的空目录，验证脚本必须来自同一 tag：

```bash
git fetch origin tag v3.5.2
git switch --detach v3.5.2

asset_dir=runtime/validation/v3.5.2-release-assets
mkdir -p "$asset_dir"
test -z "$(find "$asset_dir" -mindepth 1 -maxdepth 1 -print -quit)"

gh release download v3.5.2 \
  --repo HoneyBury/boost_gateway \
  --dir "$asset_dir" \
  --pattern 'boost-gateway-v3.5.2-linux-x64.tar.gz' \
  --pattern 'boost-gateway-v3.5.2-sbom.spdx.json' \
  --pattern 'SHA256SUMS.txt'

(cd "$asset_dir" && sha256sum -c SHA256SUMS.txt)

python3 scripts/tools/verify_release_archive.py \
  --archive "$asset_dir/boost-gateway-v3.5.2-linux-x64.tar.gz" \
  --expected-root boost-gateway-v3.5.2-linux-x64

python3 scripts/tools/verify_release_package_consumer.py \
  --archive "$asset_dir/boost-gateway-v3.5.2-linux-x64.tar.gz" \
  --expected-root boost-gateway-v3.5.2-linux-x64 \
  --image ubuntu:24.04 \
  --summary-path runtime/validation/second-runner-package-consumer-summary.json

gh attestation verify "$asset_dir/boost-gateway-v3.5.2-linux-x64.tar.gz" \
  --repo HoneyBury/boost_gateway
```

通过条件：checksum 全通过；tarball 为真实 gzip 且只有单一版本根目录；全部必需 ELF
依赖可解析；hello-world 在 `network=none` / `pull=never` 的 Ubuntu 容器内退出码为 0；
GitHub build provenance 和 SBOM attestation 可验证。
