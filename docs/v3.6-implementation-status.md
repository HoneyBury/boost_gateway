# v3.6 P0-P6 Implementation Status

更新时间：2026-07-22

P0-P6 的仓库内实现已完成，默认激活策略仍是
`blocked_until_gates_pass`。本页区分本地可复现事实与必须由固定 runner 或不可变
发布资产补齐的外部事实；后者未完成前不得宣称 production supported。

## 激活顺序

1. 在当前 exact SHA 运行 Linux `release.yml`、`sdk-distribution.yml` 和
   `debug-symbols.yml`，保存 summary、artifact ID、runner identity 与 lockfile hash。
2. 运行 `macos-arm64.yml`，只在同 SHA build/test/install/package 全通过后把
   `osx-arm64` 从 local candidate 提升为 verified candidate。
3. 为身份模块补真实 HTTPS JWKS 轮换、stale grace 和 outage drill。
4. 冻结候选 tag 后发布 GitHub Release 资产，再由独立 workflow 下载并核对
   checksum、SBOM、provenance、包消费和符号映射。
5. 只有具体版本/RID 的全部证据闭环后才更新 compatibility matrix；PyPI、
   NuGet.org、Raft protobuf writer 默认启用仍需独立激活决策。
