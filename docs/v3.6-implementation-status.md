## 激活顺序

1. 补齐 Linux ARM64 Release/R0 所需的 v3.5.3 ARM64 legacy binary 和平台化发布
   资产契约；Linux x64 dedicated SDK/debug/JWKS 已完成。
2. 在 Linux ARM64 与 macOS ARM64 分别运行三次 performance baseline，再建立不可
   互相替代的 soak/capacity 阈值。
3. 仓库变更停止后冻结一个 SHA，在三个平台刷新 Release/R0/R5、JWKS、SDK、符号与
   baseline evidence。
4. 冻结候选 tag 后发布 GitHub Release 资产，再由独立 workflow 下载并核对
   checksum、SBOM、provenance、包消费和符号映射。
5. 只有具体版本/RID 的全部证据闭环后才更新 compatibility matrix；PyPI、
   NuGet.org、Raft protobuf writer 默认启用仍需独立激活决策。
